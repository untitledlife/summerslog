ClaimPilot

AI-Powered Claims Management Agent — Proof of Concept Requirements

Prepared byConscious Engines ForEquifi / SaleMyMachine DateMarch 2026
Success criterion: A client stakeholder watches a simulated claim get handled end-to-end and says, "We need this."

1. Background

Equifi operates across the full commercial vehicle and construction equipment lifecycle — marketplace (SaleMyMachine), financing, insurance, and fleet management. The company already runs interconnected internal tools: CRM, loan tracking, PDF generation, accounting, vehicle inspection, expenses, gate pass, and yard management.

Insurance claims are a critical pain point in this ecosystem. When a financed JCB excavator or a Tata truck gets damaged on a job site, the claim process involves coordinating across 5+ parties — insurer, surveyor, service center, police (if applicable), and the RTO. This coordination is manual, slow, and opaque to the vehicle owner/operator. Delays directly translate to equipment downtime costs of INR 50,000–1,00,000+ per day for commercial equipment.

The opportunity: An AI agent that manages the end-to-end claims process on behalf of the claimant, while orchestrating data flow across Equifi's existing tools.

2. Objective

Build a working proof-of-concept that demonstrates:

  1. Conversational claims intake — a vehicle operator or fleet manager reports an incident via chat (WhatsApp-style interface), and the agent extracts all required information through natural conversation.
  2. Automated coordination — the agent tracks claim status across parties, generates required documents, and proactively nudges the claimant and internal teams when action is needed.
  3. Integration with existing systems — the agent reads from and writes to Equifi's existing tools (CRM, loan tracking, vehicle inspection, accounting, yard management), demonstrating that it orchestrates the current stack rather than replacing it.

3. Scope

3.1 In Scope (POC)

Claim type: Own Damage claim for a commercial vehicle / construction equipment (accident, on-site damage, transit damage). This is the most common and most painful claim type.

Core capabilities:

# Capability Description
1 Conversational intake Agent collects incident details through natural chat: what happened, where, when, vehicle details, policy info, photos. No forms.
2 Intelligent triage Agent determines: claim type (own damage / third-party / theft), whether FIR is needed, cashless vs reimbursement path, estimated claim value range.
3 Document checklist & tracking Personalized checklist based on claim type. Tracks which documents are collected vs pending. Sends reminders for missing items.
4 Regulatory deadline awareness Agent knows IRDAI timelines (72-hr intimation, 15-day surveyor report, 7-day insurer decision, 30-day settlement). Alerts when deadlines approach or are breached.
5 Communication drafting Generates insurer intimation letters, surveyor follow-ups, and claim status summaries in professional language.
6 System integration (simulated) Pulls vehicle and loan data from Equifi's CRM/loan tracker. Logs claim events to accounting. Updates vehicle status in yard management. Generates claim documents via PDF system.
7 Proactive status updates Agent checks claim progress and pushes updates to the claimant without being asked: "Surveyor visited today, report expected by Friday."
8 Escalation & handoff Agent identifies situations requiring human judgment (third-party injury, total loss, disputed liability) and flags them with context for a human agent.

3.2 Out of Scope (POC)

  • Live integration with any insurer's API or portal
  • Real document upload, OCR, or verification
  • Payment processing or settlement disbursement
  • Multi-user authentication or role-based access
  • Theft claims and third-party liability claims (added post-POC)
  • Mobile app — POC uses web-based chat + dashboard

3.3 Fake Data for POC

The POC will use realistic but synthetic data:

  • 3 sample vehicles: JCB 3DX backhoe loader, Tata Signa 4825.TK tipper, Ashok Leyland Dost+ LCV
  • 2 sample claims: One in-progress (mid-surveyor-stage), one initiated live during demo
  • Simulated integrations: Mock API responses from CRM, loan tracker, vehicle inspection, yard management, and accounting systems — demonstrating the data flow without requiring live connections

4.2 The Equifi Ops Team (internal dashboard)

Channel: Web dashboard (auto-refreshing)

Dashboard shows:

  • Active claims list with status indicators
  • Per-claim view: timeline, document checklist, integration activity log
  • Deadline alerts (color-coded: green / amber / red)
  • Agent activity log: every action the agent took, with reasoning
  • Handoff queue: claims flagged for human review

6. Technical Approach

  • AI backbone: Claude (Anthropic) with domain-specific system prompt containing the full Indian motor insurance claims process, IRDAI regulations, and Equifi-specific context
  • Tool layer: 6–8 function tools the agent can call (update claim state, query vehicle data, generate documents, check deadlines, flag for review, search service centers)
  • State management: Persistent claim state (JSON) per active claim — survives across conversations
  • Dashboard: Lightweight web interface pulling from the same claim state
  • Deployment: Cloud-hosted, accessible via web link for demo purposes

8. What the POC Proves

  1. The agent understands claims. It asks the right questions, knows the process, and doesn't need to be told what comes next.
  2. It orchestrates, not replaces. Equifi's existing tools become more powerful — data flows between them automatically instead of through manual copy-paste.
  3. It catches things humans miss. Regulatory deadlines, policy clause issues (depreciation, exclusions), missing documents — the agent flags them proactively.
  4. It reduces claimant effort to near-zero. The vehicle operator has a conversation. Everything else happens behind the scenes.
  5. Downtime cost drops. Faster claims = equipment back in service sooner = direct revenue impact for Equifi's customers.
